# scottycore v0.1.9 — AI backend adapters + generation schemas

Point release. Promotes the AI provider adapter layer from vendored-in-
scottystrike to first-class scottycore APIs so multiple consumer apps can
share them.

## What's new

### `scottycore.services.ai_backends.generation_schemas`

Top-level request/response models for LLM generation:
  * `Message` — role + content
  * `GenerationRequest` — unified prompt envelope passed to every adapter

Kept separate from the base `schemas.py` (config / status models) so the
lightweight health-check and settings paths don't pull in generation
dependencies.

### `scottycore.services.ai_backends.adapters`

Concrete provider adapters, all implementing the `base.BaseAdapter` protocol:

| Adapter | Provider |
|---|---|
| `azure_openai` | Azure OpenAI Service |
| `claude_api` | Anthropic direct API |
| `claude_cli` | Claude Code CLI (shell-out) |
| `dgx` | NVIDIA DGX Cloud |
| `ollama` | Ollama (local or remote) |
| `openai` | OpenAI API |
| `custom` | Pluggable HTTP adapter for custom endpoints |

Consumers import them the same way scottystrike already did:

```python
from scottycore.services.ai_backends.adapters import claude_api
from scottycore.services.ai_backends.generation_schemas import Message, GenerationRequest
```

## Upgrade

Pin bump only. No schema migration.
