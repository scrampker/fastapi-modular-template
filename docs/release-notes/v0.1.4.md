## Highlights

Small additive release — one new public helper that fixes a common
consumer-app pitfall.

### `scottycore.web.install_static(app, ...)` — new helper

Consumer apps that build their own FastAPI instance (rather than calling
`scottycore.main.create_app`) must mount scottycore's shipped static assets
themselves. Until now each app had to derive the install path manually, and
at least one app (scottydev) shipped with an empty local static directory
that shadowed the lookup — causing the post-login dashboard at
`/c/{slug}/dashboard` to render a blank slate background because no
JavaScript or CSS loaded.

The new helper encapsulates the mount:

```python
# Consumer app main.py
from scottycore.web import install_static

install_static(app)                                     # mount /static from pip-installed scottycore
install_static(app, overlay_dir="/path/to/app/static")  # + overlay at /static-app
```

Signature:

```python
install_static(
    app,
    *,
    overlay_dir: str | None = None,
    mount_path: str = "/static",
    overlay_mount_path: str = "/static-app",
    name: str = "static",
) -> None
```

scottycore's own `create_app()` has been refactored to use the helper, so
the two code paths stay in sync.

## Adoption Guide

### Drop the per-app copy of `web/static/`

Apps scaffolded from scottycore commonly have a `web/static/` directory that
is a frozen copy of scottycore's shipped assets. Once on v0.1.4 you can delete
that copy and use the helper instead:

**Before**
```python
# app/main.py
_static_dir = os.path.join(os.path.dirname(__file__), "web", "static")
if os.path.isdir(_static_dir):
    app.mount("/static", StaticFiles(directory=_static_dir), name="static")
```

**After**
```python
from scottycore.web import install_static
install_static(app)  # overlay_dir=... if you ship app-specific static
```

Benefits:
- No manifest/JS drift as scottycore evolves
- Smaller app repo (no duplicated binary assets)
- Consistent behavior across the Scotty app family

## No breaking changes

scottycore's own `create_app()` continues to mount `/static` as before; the
internal implementation simply delegates to the new helper. Apps that don't
adopt the helper keep working.
