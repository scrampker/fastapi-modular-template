# scottycore v0.1.10 — pip-install data_dir fix

## Bug fix

When scottycore is pip-installed into site-packages (as opposed to a
checkout), `create_app()` resolved the request-logging data directory
relative to the installed module path:

    /usr/local/lib/python3.12/site-packages/data

which is not writable under the container's non-root user, crashing the
app at startup with `PermissionError`.

`create_app()` now resolves `data_dir` in this priority order:

1. `SCOTTYCORE_DATA_DIR` environment variable (explicit override)
2. `<scottycore-parent>/data` — only if the parent directory is writable
   (i.e. you're running from a source checkout)
3. `/app/data` — the Dockerfile default across all consumer apps

## Upgrade

Pin bump only. Consumer apps that already have `/app/data` writable (which
they all do — the shared Dockerfile pattern creates it) just work.

```
"scottycore @ git+https://github.com/scrampker/scottycore.git@v0.1.10"
```
