# scottycore v0.1.8 — auto-update subsystem

## What's new

Every consumer app now has a first-class mechanism for keeping its
scottycore pin current. Three policies, toggled via the scottycore settings
table so the choice can be made either in the local app's UI **or** pushed
down from ScottyDev:

| Mode | Behaviour |
|---|---|
| `off` | Do nothing. |
| `notify` (default) | Write `scottycore.update.pending` to settings; UI surfaces a banner. |
| `auto` | Bump `pyproject.toml` pin, drop `.scottycore-auto-update-requested` flag, exit 75. |

### Components shipped

* `scottycore.services.auto_update.AutoUpdateService` — the poll-cycle
  engine. Read-only w.r.t. the runtime container: bumping the pin happens
  on disk and a host-side runner picks up the rebuild flag separately so
  the poller needs no docker-in-docker privileges.
* `scottycore-update-check` — new console script, consumed by a systemd
  timer. Exit 0 = nothing to do, 75 = please rebuild, 1 = error.
* Systemd unit templates under
  `scottycore/services/auto_update/systemd/`:
  * `scottycore-update-check@.service`
  * `scottycore-update-check@.timer` (every 60 s, 30 s accuracy)

### Rendezvous settings keys

* `scottycore.update.mode` → `"off"|"notify"|"auto"` (default `"notify"`)
* `scottycore.update.pending` → `{"current","latest","ts"}` written by the
  poller when a newer version is detected in notify mode

### Version detection

Uses GitHub's `/releases/latest` first; falls back to `/tags` with PEP440
version-sort when the repo has no named releases. Both GitHub and Forgejo
APIs are compatible via the `--forge-api` flag on the CLI.

## Tests

8 new unit tests in `tests/services/auto_update/` covering: up-to-date,
off/notify/auto modes, missing pyproject, malformed pin, 404 fallback to
tags, unknown-mode default-to-notify.

## Upgrade

Pin bump only. No schema migration. To activate:

1. Bump the pin in `pyproject.toml`: `scottycore @ git+…@v0.1.8`
2. On each app host:
   ```
   cp scottycore-update-check@.service /etc/systemd/system/
   cp scottycore-update-check@.timer   /etc/systemd/system/
   systemctl enable --now scottycore-update-check@<app-slug>.timer
   ```
3. Toggle the mode via settings (default is `notify`).
