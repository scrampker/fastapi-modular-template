# scottycore v0.3.1 — GitRepoSink

Patch release. Adds a fifth backup sink — forge-agnostic git-repo
storage with LFS support — alongside the existing four
(`local_disk`, `scottydev`, `remote_node`, `download`).

## What's new

### `GitRepoSink` — backups as git commits

Works against any git remote (Forgejo, GitHub, GitLab, Azure DevOps,
Gitea, raw SSH) using the git credential helper / SSH keys that are
already configured on the host. No per-forge adapter, no auth code.

Each `put()` lands a commit; `list_snapshots()` walks the working tree
and reads `.meta.json` sidecars; `delete()` lands a `git rm` commit.
Non-fast-forward pushes auto-recover with a `pull --rebase` + retry.
Network failures degrade gracefully — commits stay local and push on
the next successful operation.

LFS is enabled by default for `*.gpg`, `*.tar`, `*.tar.gz` patterns.
Disable with `lfs_enabled=False` for small deployments or forges that
don't have LFS configured.

### Path templates

Storage layout is fully user-configurable via `path_template`:

```python
GitRepoSink(
    repo_url="https://forgejo.example.com/ops/fleet-backups.git",
    local_clone_dir="/app/data/backups-git",
    path_template="snapshots/{app_slug}/{scope}/{tenant_slug}",
    # or flat: "{app_slug}-{scope}-{timestamp}"
    # or per-tenant-first: "tenants/{tenant_slug}/{app_slug}/{scope}"
)
```

Template vars: `{app_slug}`, `{scope}`, `{tenant_slug}`, `{kind}`,
`{timestamp}`. Empty segments collapse (platform-scope bundles don't
leave trailing `/` with an empty `{tenant_slug}`).

### CLI

New flags on `scottycore-backup {export,restore,verify}`:

```
--sink git_repo
--repo-url URL           # required for git_repo
--clone-dir PATH         # default /app/data/backups-git
--branch NAME            # default 'backups'
--path-template TEMPLATE # default snapshots/{app_slug}/{scope}/{tenant_slug}
--no-lfs                 # disable LFS
```

### REST + scheduler

`POST /api/v1/backup/schedules` and the in-process scheduler both
accept `sink_type: "git_repo"` with `sink_config` keys
`repo_url`, `branch`, `path_template`, `lfs_enabled`, and optional
`clone_dir`.

## Why it's useful

- **Free versioning** — `git log` is the snapshot index; diffs show
  exactly when bundles were added, rotated, or deleted.
- **Offsite by default** — push to a remote on different hardware /
  cloud / region.
- **Access control via forge** — one repo-level permission boundary
  covers the entire backup store. Revoke a stolen key by rotating
  on the forge.
- **Cheap retention** — LFS-backed blobs don't bloat the git history;
  prune with `git lfs prune` (destructive, opt-in via a future
  `scottycore-backup gc` command).

## Scottydev fleet integration

scottydev v0.3.1 adds a matching `POST /api/v1/backup/fleet-repo`
admin endpoint that stores a fleet-wide repo URL + branch + template
in the scottycore settings table. Managed schedules that scottydev
creates on consumer apps automatically inherit the fleet URL +
`{app_slug}` sub-path prefix.

Customers who prefer per-app repos can still override by editing the
schedule's `sink_config` individually.

## Tests

- 18 new tests for `GitRepoSink` against a local bare repo (no network)
- 7 new tests for the scottydev `fleet_repo` helper
- No regressions in the existing 210 + 217 test suites

## Compatibility

Fully backwards-compatible. Existing sinks are untouched. The git_repo
sink is opt-in — apps see no change unless they explicitly configure it.
