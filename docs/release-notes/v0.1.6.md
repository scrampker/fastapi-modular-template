# scottycore v0.1.6 — chaos fixes for backup subsystem

Point release. Follow-on to v0.1.5 after a hostile/edge-case test pass.

## Bug fixes

* **`LocalDiskSink` concurrent write race** — two simultaneous `put()` calls
  derived the same `.part` filename; one writer's rename raced the other's
  final rename and one side saw `FileNotFoundError`. Temp name is now
  unique per-write (`{pid}.{id(blob):016x}.part`).
* **Passphrase newline handling** — GPG silently strips trailing newlines
  from passphrase files, so `"pw\n"` and `"pw"` produced the same ciphertext
  but different fingerprints, leading to a restore-key confusion. Passphrase
  is now normalised (`rstrip("\r\n")`) consistently before fingerprint
  computation and GPG invocation.
* **`_build_sink(local_disk, {"root_dir": null})` crash** — `dict.get(key,
  default)` returns `None` when the key is present with a null value, so
  `Path(None)` threw `TypeError` (500) instead of falling back to the
  hardcoded default. Fixed with `or` fallback.
* **`UserContext.id` → `user_id`** — backup REST handlers referenced
  `user.id`, which doesn't exist on `UserContext`; the export/restore
  endpoints 500'd when called through the live stack. All six call sites
  now use `user.user_id`.

## Tests

**161 + 1 skipped** on this build.
* 80 chaos tests covering passphrase edge cases (empty/long/unicode/newlines/
  shell-metachars), tampered bundles (bit-flip/truncate/append), path
  traversal, symlink escape, concurrent writes, tampered manifests,
  SQL-injection-shaped locators, disk errors.
* All pre-existing v0.1.5 backup tests still green.
